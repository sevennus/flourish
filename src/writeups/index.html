✦ Flourish — writeups

Every bug that cost more than a minute, with the evidence used to find it.

The standing rule on this VM is that a fix isn't done until there's a writeup: the symptom, how it was found, the root cause, the options, and the verification — with the artifacts that actually settled it, not just prose about them. These are Flourish's own. They live in this repo and ship with it, so the app carries its own history rather than borrowing SimJim's.

lightning never reached the bottom, and three separate caps agreed it shouldn't
2026-07-16 · renderer.js, flourish.js, effects.js, tools/lightning-probe.js
"lightning didnt read the bottom" — and it couldn't, on any page. Three reasonable limits stacked into 90px of spread on a 700px screen. The decisive one is the fix for the last bug: stratifyAnchors cured apophenia's collinear sampling and carried a top-of-screen bias out the other side, because max below the line count silently means "the top max lines". Invisible while apophenia (max=14) was its only caller. The probe had been printing anchors: 4 / struck: 3 for weeks.

every reply died after its first word, and the prompt box never came back
2026-07-16 · fixed in 87ff19d · renderer.js
Two functions named appendText in one scope. Declarations hoist, the later one silently wins, and the throw landed inside the typewriter's rAF loop — which is why one line caused both symptoms at once. 88 green tests never touched it.

⚠ Reading the SHAs

Flourish moved out of the SimJim tree on 2026-07-16 into its own repo (github.com/sevennus/flourish). The move used git subtree split, so all 21 commits kept their content and messages but were rewritten to new SHAs.

Writeups above were authored before the split and cite SimJim SHAs. Those commits still exist in SimJim's history — only the files moved — so the references remain resolvable there. Writeups from the split onward cite Flourish SHAs. Both are valid; they just point at different repos. Where these are served

Static files under src/, served by server.js at /flourish/writeups/index.html — behind the same tailnet-only fence as the app itself, because it is the same location block. No server.js change was needed to publish them, which also means no restart: editing a writeup is a browser reload.

Directory URLs don't resolve — serveStatic has no index fallback, so link to <slug>/index.html explicitly, as the cards above do.
