The grid register, volume II: ten cheeky effects

Branch main · flourish.js, effects.js, prompt.js, tools/ascii-probe.js, tools/fx-shots.js, test/ascii.test.js · 2026-07-17

The ask

"i want to see 10 more ascii art effects like that. dont be afraid to be creative and cheeky."

"Like that" = the grid register: effects painted into the terminal's own character cells, built out of the prose already on screen, mutating nothing. Ten more, cheekier — mostly arcade and screensaver gags, each one doing something to your words.

Same architecture as volume I: each fact is a pure seeded planner in flourish.js (node --test pins it), and the geometry, motion and incorporation live in effects.js against a grid it can't see. Motion runs in _step<Name> from the update loop so it gets real dt; nothing mutates the DOM, so every eaten pellet, abducted word and melted line is a knockout drawn on canvas over untouched text — whole again the instant the scene is reaped. The one genuine algorithm, Conway's step, is pure and exported, and the tests pin it against a blinker and a block.

Verification — and the three things the harnesses (and eyes) caught

The order this repo insists on: plan pure, probe the paint, then look anyway. 218 unit tests (Conway against textbook oscillators, every planner sane at seven seeds, the sprites rectangular); ascii-probe extended with a faithfulness oracle per new scene — a snake head @, a saucer, a ghost, fish — verdict 25/25 planned / painted / faithful / reaped (full output); then every one read as a screenshot.

  1. The probe threw twice on a field-name collision. planInvaders returns rows as a number, and the probe indexed S.plan.rows.map(...); the life scene carried a numeric rows, and the reveal-timing oracle indexed S.rows[0].length. Both threw "Script failed to execute" with no hint which scene. A one-line process.stdout.write(name) localised it to life in one run. Fixed by guarding every index with Array.isArray and renaming life's dims to lcols/lrows so a number can never sit where an array is expected. A shared field name across effects is a collision waiting for the next author.
  2. quake built an impossible spire. The heap stacked per-origin-column with no ceiling, so a column that held forty characters towered — measured at 394px above the top of the screen, 40% of its draws off-frame. The probe flagged it (off-screen > 0, and quake is not a motion-exempt effect). Capped the heap at seven rows: past the ceiling, characters pile on each other instead of stacking further. above: 0, below: 0 at every sample after.
  3. the aquarium photographed empty, and only the screenshot knew. Fish spawned up to a full screen-width off one edge and swam at ~0.05px/ms — they never reached the tank inside a shot. The probe passed it: a fish string drawn off-screen still counts as "own content", and the motion exemption forgave the off-screen draws. This is the exact failure this repo keeps re-learning — a green oracle over a blank frame — and the fix for it is the same as always: look. Fish now spawn spread across the tank.

Vocabulary: 84 effects (POINT_EFFECTS 57, ASCII_EFFECTS 25, GRID_EFFECTS 17). prompt.js and demo.js are baked into the server at startup, so the new names go live on the next systemctl restart flourish — the user's command to run, not this session's.
