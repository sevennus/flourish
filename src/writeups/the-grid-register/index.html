The grid register: effects that paint themselves out of the prose

Branch main · flourish.js, effects.js, renderer.js, prompt.js, demo.js, tools/ascii-probe.js, tools/fx-shots.js · 2026-07-17

The ask

"with the ascii art, it shouldn't be line art just floating in the window. it should be ascii art on the screen, incorporating the characters on the screen. The skull should animate, it's jaw should move. there should be like a spinning wireframe sphere / prism. also should make an effect where a small cat pops out of the text and can walk along the lines of adjacent text, even falling to lower lines of text. id like more effects that have a lot of colors and shaped and such but that they try to emulate using the character grid in the console."

Four asks, one architectural answer: the terminal's text has a character grid, and effects should live IN it — same font, same cells, same rows as the prose — and be built out of the characters already on screen, not painted over them at some unrelated size.

The grid layer

The division of labour stays where every prior effect put it: the renderer owns the text, so the renderer measures; the engine gets numbers. Two new measurement functions in renderer.js:

  • gridSnapshot() — walks the visible transcript in runs (maximal non-space slices of a text node): one Range rect per run, a few hundred reads for a full screen, the same order lightning already pays. Monospace means every character inside a run sits at left + k·cellW exactly. Returns cell metrics (advance width measured off the longest runs, median not mean), the font, and a map from grid cell → the real character there with its true measured position — because the 10px paragraph margins mean the uniform grid and the actual rows drift apart down the page, and painting a "replaced" glyph 3px off its DOM twin reads as ghosting.
  • linePlatforms() — the same runs, merged per row into ledge segments. Gaps wider than a couple of cells split a row: the end of a paragraph is a real edge to fall off. This is the cat's level geometry.

Both are exposed on window.Flourish for the harnesses, for exactly the reason strikeTargets is: a harness that fires these bare is photographing a branch no reader ever reaches.

Three rules the whole register obeys:

  • Incorporation. Wherever an effect's ink lands on a real character, it paints that character — the prose lights up, recolours, or burns as part of the picture. Where the page is empty, the effect brings its own glyphs.
  • No fallback. A grid effect fired without a grid paints nothing. This repo has now shipped an invisible-but-photogenic fallback three separate times (apophenia, salvage, the probe's font regex); blank is a failure someone sees.
  • Nothing is destroyed. Every recolour, knockout and veil is canvas over untouched DOM. The prose walks out unharmed when the scene fades — the one standing rule about Jim and text that erodes.

Scenes that ride on text also ride with it: each carries scrollDrift and translates by the scroll delta per frame, the same aliasing salvage pays for.

The seven older panes (wardial, sniffer, trace, daemon, portscan, overflow — not crack or gibson, which are deliberately cinematic) take the snapshot as an upgrade: with it they snap to the text's rows and columns and print in the page's own face, like terminal output instead of a floating window. Bare, they keep their old path.

How it was verified (and what the harnesses caught this time)

The pattern from the ASCII-scenes build, kept on purpose: plan pure, probe the paint, then look anyway.

  • Unit tests (215, all green): every planner at seven seeds — the jaw shut at rest and at death, never past full gape, chomps non-overlapping; every wireframe a closed graph inside the unit ball; rastered segments connected (a gap is a dashed stroke); the plasma field bounded; firewall heat bounded and cooler with height; cat frames rigid and mirroring an involution. Plus the prompt-parity test, which is what forces a new name into prompt.js before it can ship.
  • ascii-probe, extended to fire the way applyEvents fires: real prose, real snapshot, and it throws if the grid comes back thin. New oracles per scene: plasma's canvas strings must BE the transcript's own words; a wireframe frame must hold two distinct stroke glyphs (one glyph = a rotation degenerated to a line); the cat must move across a 4s window. Verdict: 15/15 planned, painted, on-screen, faithful, reaped (full output).
  • fx-shots: eleven new frames over real prose, then actually read as images — the step that has caught what counts cannot (gibson's smear). This time looking caught nothing the probe missed; the probe caught what looking missed (below).
  • npm run smoke:web against the running server: green.

The three things the harnesses caught

  1. The probe couldn't see a banner made of blocks. It counted fillText and strokeRect; the grid banner is fillRect blocks plus knockout glyphs, so a correct banner probed as painted 0. The tap now counts fills, per frame. Same lesson as the 126px font regex: when a working feature fails the harness, suspect the harness first.
  2. The cat's motion check was narrower than a sit. Sits last up to ~2.2s; the check sampled inside one and printed "the cat never moved" about a working cat. The window is now 4s of samples — and the tracing that proved the cat innocent also showed it sitting every ~2.5s, half its life parked. sitP tuned 0.0004→0.00018: a cat with somewhere to be.
  3. One capture race. A single fx-shots cat frame came back catless with a smeared compositor band; three independent reproductions (state trace, draw-call tap, fresh screenshot) showed the cat present and painting. xvfb software-GL capturePage can catch a frame mid-composite; noted here so the next blank single frame gets re-shot before it gets "fixed".

Decisions worth recording

  • parseArgs grew a words pass-through so wireframe can hear sphere · prism · cube without every effect growing its own parser. Unknown words cost nothing.
  • Plasma recolours per chunk (≤7 chars), not per character — one fillText per chunk keeps a full screen of prose near gibson's measured frame budget, and colour resolution at 7 characters is invisible at plasma's spatial frequencies.
  • The canvas baseline is derived from the measured run height (rh, median), not from the font size — that's what makes a canvas glyph land exactly on its DOM twin, and it is why the plasma shot shows no doubling.
  • Scale args don't resize grid scenes — a grid effect at 1.3× cell size is no longer on the grid. wireframe maps scale to radius; the others ignore it.
